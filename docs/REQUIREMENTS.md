# MySerial 需求规格说明书

> 版本：0.1 · 状态：阶段 0 已完成脚手架
> 定位：Windows 端个人串口调试工具，对标 [HTerm](https://www.der-hammer.info/pages/terminal.html) 并补齐其三大短板。

---

## 1. 背景与目标

HTerm 是一款优秀的串口调试工具，但存在三个长期未解决的痛点，本项目的核心目标就是**在这三点上全面超越 HTerm**，同时保留 HTerm 已有的实用功能集。

### 1.1 三大核心改进（vs HTerm）

| 编号 | 痛点 | 目标 |
|------|------|------|
| **R1** | HTerm 不支持 ANSI 彩色转义 | 完整解析 ANSI SGR 序列：前景/背景色、加粗、斜体、下划线、闪烁、反色、256 色、24 位真彩色，以及光标控制序列 |
| **R2** | HTerm 不支持中文界面，串口也不能正确收发 UTF-8 中文 | 串口收发**原生 UTF-8 中文**（跨 read 边界不产生半个汉字）；界面提供**中文/英文**双语切换 |
| **R3** | HTerm 不支持串口断开后自动重连 | 监测读错误/设备消失，按配置自动重连；USB 热插拔实时刷新端口列表 |

### 1.2 成功标准

- 用 ESP32/Arduino 持续打印 `"\x1b[31m红色测试\x1b[0m \x1b[32m绿色\x1b[0m"` 能正确显示彩色中文。
- 中途拔插 USB，应用不崩溃，端口列表自动更新，启用自动重连后能自行恢复数据流。
- 115200 波特率持续输入 24 小时，无丢字、无内存泄漏。

## 2. 功能需求

### 2.1 三大核心改进（必做）

#### R1. ANSI 彩色转义
- **前端 xterm.js 直接渲染**：`term.write(utf8String)` 自动解析 SGR。
- 支持序列类型：
  - SGR（Select Graphic Rendition）：前景/背景 16 色、256 色、24 位真彩色、加粗、暗淡、斜体、下划线、慢/快闪烁、反色、隐藏、删除线、重置。
  - 光标控制：CR `\r`、LF `\n`、BS `\b`、TAB `\t`、BEL `\x07`、光标上下左右移动、清屏、清行。
- **开关**：是否启用彩色解析（关闭时以原始字符显示，便于排查裸字节）。

#### R2. 中文 UTF-8 收发 + 双语界面
- **接收解码（关键）**：Rust 读线程维护 `Vec<u8>` 缓冲，每次 `read` 追加后**只截取到完整 UTF-8 字符边界**，残留的不完整多字节序列留到下次 read 拼接，再通过 IPC event 推送已解码字符串。杜绝"半个汉字"乱码。
- **发送编码**：前端输入按 UTF-8 编码为字节序列发送。
- **换行处理**：
  - 发送时可选追加 `\n` / `\r\n` / `\r` / 无。
  - 接收时可选统一渲染 `\r\n` / `\r` / `\n` 为换行。
- **界面 i18n**：svelte-i18n 实现**中文（zh-CN）/ 英文（en-US）**切换，全 UI 文案覆盖；语言偏好持久化到 localStorage。

#### R3. 断线自动重连
- **触发条件**：读线程返回 IO 错误、EOF、或端口从枚举列表中消失。
- **状态机**：
  ```
  Connected ──(读错/EOF/设备消失)──► Reconnecting ──(成功)──► Connected
                                          │
                                          └──(达最大次数)──► Lost
  ```
- **配置项**：
  - `enabled`：是否启用自动重连（默认 true）。
  - `interval_ms`：重连间隔（默认 1000）。
  - `max_attempts`：最大尝试次数，0 = 无限（默认 0）。
  - `on_port_gone`：端口消失时行为（`prompt` 提示重选 / `wait` 等待同名重现，默认 `prompt`）。
- **前端反馈**：
  - 状态指示器实时显示 `已连接 / 重连中(N/Max) / 连接丢失`。
  - 重连成功后无缝恢复数据流；用户可手动中断重连。
- **USB 热插拔**：端口列表监听变化，插拔事件实时刷新下拉并提示。

### 2.2 对标 HTerm 基础功能

#### R4. 串口参数配置
- 端口选择（下拉 + 手动输入）。
- 波特率：常用预设（9600/19200/38400/57600/115200/230400/460800/921600）+ 自定义输入。
- 数据位：5 / 6 / 7 / 8。
- 校验位：None / Even / Odd / Mark / Space。
- 停止位：1 / 1.5 / 2。
- 流控：None / Software (XON/XOFF) / Hardware (RTS/CTS)。

#### R5. 端口枚举与热插拔
- 列表显示：端口名（COMx）、友好描述（如 "USB Serial Device (COM3)"）、VID/PID。
- 热插拔：USB-CDC 设备插拔时实时刷新下拉，新增/移除有视觉提示。

#### R6. 显示模式（接收区）
- ASCII 文本模式。
- Hex 十六进制模式（每字节空格分隔）。
- Dec 十进制模式。
- Bin 二进制模式。
- **ASCII + Hex 双列对照**（左 ASCII 右 Hex，最常用）。
- 模式可运行时切换，不丢失已接收数据。

#### R7. 发送区
- 单次发送：输入框 + 发送按钮（回车可配置为发送）。
- 循环发送：开启后按设定间隔（ms）重复发送当前内容。
- 发送历史：最近 N 条（默认 20），点击即重发。
- 发送宏：命名预设命令列表（如 "复位设备" → `"reset\n"`），一键发送；宏可增删改、持久化。
- 发送内容支持 Hex 输入（如 `0x41 0x42` 或 `41 42`）。

#### R8. 控制信号
- 输出信号（可手动控制）：RTS、DTR 开关。
- 输入信号（只读状态灯）：CTS、DSR、CD (DCD)、RI，实时刷新。

#### R9. 时间戳
- 每行起始可选添加时间戳：
  - 关闭。
  - 绝对时间 `HH:mm:ss.fff`。
  - 相对会话开始 `+0.123s`。
- 时间戳颜色与数据区分。

#### R10. 接收区操作
- 清空、暂停/继续滚动、复制选中、全选。
- 搜索（基于 xterm-search addon）。

#### R11. 换行处理
- 见 R2 换行子项。

#### R12. 统计信息
- 接收字节总数、发送字节总数。
- 当前速率（RX/TX bytes/s，滑动窗口）。
- 会话时长（连接开始至今）。
- 错误计数（校验错、帧错、溢出错，若底层暴露）。

### 2.3 数据记录与导出

#### R13. 实时日志
- 可选开启：所有收发数据带时间戳写入日志文件。
- 按会话（Session）分文件，文件名 `myserial_<session>_<timestamp>.log`。
- 日志目录可配置（默认 `~/Documents/MySerial/logs/`）。

#### R14. 导出
- 当前会话导出为：
  - `.txt`（纯文本）。
  - `.csv`（时间戳, 方向, 数据）。
  - `.hex`（十六进制 dump）。
- 格式跟随当前显示模式或单独选择。

### 2.4 多标签 / 多串口

#### R15. 多 Tab 会话
- 每个 Tab 一个独立串口连接与完整配置。
- 可同时打开多个串口（如 COM3 + COM5 同时工作）。
- Tab 可重命名、关闭；关闭前若已连接需二次确认。
- 各 Tab 状态完全独立（统计、历史、宏互不干扰）。
- 新建 Tab 时复制最近 Tab 的参数作为默认（可设置）。

## 3. 非功能需求

| 编号 | 维度 | 要求 |
|------|------|------|
| **N1** | 性能 | 115200 持续输入下 UI 不卡顿；目标 ≥1Mbps 不丢字（xterm.js 批量渲染 + Rust 抢读） |
| **N2** | 稳定性 | 异常断连不崩溃；24 小时连续运行内存增长 < 20MB |
| **N3** | 美观 | Win11 Fluent 设计语言；深色/浅色主题切换；自定义终端配色 |
| **N4** | 易用 | 常用操作一键直达（连接/断开/清空/发送）；快捷键支持（Ctrl+Enter 发送、Ctrl+L 清屏等） |
| **N5** | 可维护 | 模块化（SessionManager / PortEnumerator / Logger 解耦）；关键逻辑单元测试覆盖 |
| **N6** | 体积 | 安装包 < 30MB（Tauri 优势） |
| **N7** | 兼容 | Windows 10 1809+ / Windows 11；自动处理 WebView2 缺失 |

---

## 4. 范围外（Out of Scope，v1 不做）

- 跨平台（macOS/Linux）—— Tauri 架构已预留，但 v1 只测 Windows。
- SSH / Telnet / 网络串口（TCP/UDP）—— 仅物理/虚拟 COM 口。
- 脚本/插件系统 —— 宏已能满足大部分需求。
- 串口协议解析器（Modbus 等）—— 未来可加，v1 不做。
- 数据可视化（波形图等）。
